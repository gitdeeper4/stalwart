# Contributing to STALWART

We welcome your contributions! This guide explains how to contribute to the project.

## Ways to Contribute

### 1. Report Issues or Suggest Improvements
- Open a new Issue on GitHub
- Describe the problem or suggestion in detail
- Add a label (bug, enhancement, question)

### 2. Code Contributions
1. Fork the project
2. Create a new branch (`git checkout -b feature/amazing-feature`)
3. Implement your changes
4. Test your changes
5. Submit a Pull Request

### 3. Documentation Contributions
- Improve README.md
- Add examples
- Translate documentation
- Write tutorials

### 4. Scientific Contributions
- Propose new physical indicators
- Review methodology
- Suggest improved measurement methods
- Share test data

## Code Requirements
- Follow PEP 8 standards
- Write tests for new code
- Document functions and classes
- Use type hints

## Development Process
1. **Discuss**: Discuss the idea in an Issue first
2. **Plan**: Define requirements and design
3. **Develop**: Write code with tests
4. **Review**: Review code together
5. **Merge**: Merge changes after approval

## Acceptance Criteria
- Code works and passes tests
- Documentation is updated
- No conflicts with main branch
- Quality standards met

## Communication
- GitHub Issues: For problems and suggestions
- GitHub Discussions: For general discussions
- Email: [Project email]
